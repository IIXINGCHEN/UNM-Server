API 使用示例

GET /search

根据关键字搜索歌曲。

参数:
  • name: (必需) 搜索关键字（歌曲名、歌手名、专辑名）。
  • source: (可选) 指定音乐源 (如 netease, kugou, kuwo, migu 等)。默认为 netease。注意：部分源可能失效。
  • count: (可选) 返回结果数量，默认为 20。
  • pages: (可选) 返回结果页码，默认为 1。
示例 URL: 
https://[你的域名]/search?name=[关键字]&source=kugou
成功响应:

返回一个包含歌曲列表的 JSON 对象。每个歌曲对象包含 id (曲目ID), name, artist, album, pic_id, lyric_id, source 等字段。

如果原始请求的源失败，服务器会自动尝试回退到 kugou 源，响应消息会注明。

GET /url

获取指定歌曲 ID 的播放链接。

参数:
  • id: (必需) 歌曲的 Track ID (可通过 /search 接口获取)。
  • source: (可选) 指定音乐源。默认为 netease。
  • br: (可选) 请求的音质 (128, 192, 320, 740 (FLAC), 999 (FLAC))。默认为 999。
示例 URL: 
https://[你的域名]/url?id=[歌曲ID]&source=kuwo&br=320
成功响应:

返回包含歌曲链接信息的 JSON 对象，主要字段有 url, br (实际音质), size, source。如果配置了代理且链接需要代理，还会包含 proxyUrl。

如果原始请求的源失败，服务器会自动尝试回退到 kugou 源，响应消息和 source 字段会注明。

GET /pic

获取指定专辑图 ID 的图片链接。

参数:
  • id: (必需) 专辑图的 Pic ID (可通过 /search 接口获取)。
  • source: (可选) 指定音乐源。默认为 netease。
  • size: (可选) 请求的图片尺寸 (300 或 500)。默认为 300。
示例 URL: 
https://[你的域名]/pic?id=[专辑图ID]&source=kugou&size=500
成功响应:

返回包含专辑图链接的 JSON 对象，主要字段为 url。

如果原始请求的源失败，服务器会自动尝试回退到 kugou 源，响应消息会注明，响应数据中会包含 source: 'kugou'。

GET /lyric

获取指定歌词 ID 的歌词内容。

参数:
  • id: (必需) 歌词的 Lyric ID (通常与歌曲 ID 相同，可通过 /search 接口获取)。
  • source: (可选) 指定音乐源。默认为 netease。
示例 URL: 
https://[你的域名]/lyric?id=[歌词ID]&source=qq
成功响应:

返回包含歌词内容的 JSON 对象，主要字段有 lyric (LRC 歌词) 和 tlyric (翻译歌词，可能不存在)。

如果原始请求的源失败，服务器会自动尝试回退到 kugou 源，响应消息会注明，响应数据中会包含 source: 'kugou'。

GET /match

（注意: 此接口直接调用 @unblockneteasemusic/server 核心库进行匹配，不使用 GDStudio API，无回退逻辑）。根据网易云歌曲 ID 尝试从多个音源匹配可播放链接。

参数:
  • id: (必需) 网易云歌曲的 ID。
  • server: (可选) 指定要尝试匹配的音源列表（逗号分隔，如 kuwo,migu,qq）。默认为一组内置音源（见 routes/index.js 代码）。
示例 URL: 
https://[你的域名]/match?id=[网易云歌曲ID]&server=kuwo,qq
成功响应:

返回包含匹配结果的 JSON 对象，其中 `data` 字段包含匹配到的 `url` 等信息。如果配置了代理且链接需要代理，还会包含 `proxyUrl`。
